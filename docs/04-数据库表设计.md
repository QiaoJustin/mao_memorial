## 4. 表结构详细设计

### 4.1 eras（年代分类表）

**用途**：存储6个历史年代分类信息。

```sql
CREATE TABLE `eras` (
  `id` VARCHAR(32) NOT NULL COMMENT '年代ID，如 youth/revolution',
  `name` VARCHAR(50) NOT NULL COMMENT '年代名称，如 求学探索',
  `period` VARCHAR(50) NOT NULL COMMENT '时间范围，如 1918—1926',
  `description` TEXT COMMENT '年代描述',
  `icon` VARCHAR(50) COMMENT '图标名称',
  `color` VARCHAR(20) COMMENT '主题色，如 #8B0000',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序序号',
  `node_count` INT NOT NULL DEFAULT 0 COMMENT '节点数量（冗余字段）',
  `is_active` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_sort_order` (`sort_order`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='年代分类表';
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | VARCHAR(32) | 是 | 主键，语义化ID |
| name | VARCHAR(50) | 是 | 年代名称 |
| period | VARCHAR(50) | 是 | 时间范围 |
| description | TEXT | 否 | 年代详细描述 |
| icon | VARCHAR(50) | 否 | 图标标识（对应前端图标库） |
| color | VARCHAR(20) | 否 | 主题色（十六进制） |
| sort_order | INT | 是 | 排序序号，从小到大 |
| node_count | INT | 是 | 该年代下节点数量 |
| is_active | TINYINT(1) | 是 | 是否启用：1启用 0禁用 |

### 4.2 timeline_nodes（时间节点表）

**用途**：存储毛主席生平的100个重要时间节点。

```sql
CREATE TABLE `timeline_nodes` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `date` VARCHAR(50) NOT NULL COMMENT '显示日期，如 1949年10月1日',
  `date_sort` DATE NOT NULL COMMENT '排序日期，如 1949-10-01',
  `year` INT NOT NULL COMMENT '年份，如 1949',
  `era_id` VARCHAR(32) NOT NULL COMMENT '年代分类ID',
  `title` VARCHAR(200) NOT NULL COMMENT '事件标题',
  `description` TEXT NOT NULL COMMENT '事件描述',
  `historical_context` TEXT COMMENT '历史背景',
  `thumbnail_url` VARCHAR(500) NOT NULL COMMENT '缩略图URL',
  `photo_count` INT NOT NULL DEFAULT 1 COMMENT '照片数量',
  `view_count` INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '浏览量',
  `like_count` INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '点赞数',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序序号',
  `is_published` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否发布：1已发布 0草稿',
  `is_featured` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否精选',
  `seo_title` VARCHAR(200) COMMENT 'SEO标题',
  `seo_description` VARCHAR(500) COMMENT 'SEO描述',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '软删除',
  PRIMARY KEY (`id`),
  INDEX `idx_date_sort` (`date_sort`),
  INDEX `idx_year` (`year`),
  INDEX `idx_era_id` (`era_id`),
  INDEX `idx_sort_order` (`sort_order`),
  INDEX `idx_is_published` (`is_published`),
  INDEX `idx_is_featured` (`is_featured`),
  INDEX `idx_view_count` (`view_count` DESC),
  FULLTEXT INDEX `ft_search` (`title`, `description`, `historical_context`) WITH PARSER ngram,
  CONSTRAINT `fk_nodes_era` FOREIGN KEY (`era_id`) REFERENCES `eras`(`id`) ON UPDATE CASCADE ON DELETE RESTRICT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='时间节点表';
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | 是 | 自增主键 |
| date | VARCHAR(50) | 是 | 显示用日期字符串 |
| date_sort | DATE | 是 | 用于排序的日期 |
| year | INT | 是 | 年份（冗余，便于按年查询） |
| era_id | VARCHAR(32) | 是 | 外键，关联 eras.id |
| title | VARCHAR(200) | 是 | 事件标题 |
| description | TEXT | 是 | 事件详细描述 |
| historical_context | TEXT | 否 | 历史背景补充 |
| thumbnail_url | VARCHAR(500) | 是 | 缩略图URL |
| photo_count | INT | 是 | 照片数量 |
| view_count | INT UNSIGNED | 是 | 浏览量 |
| like_count | INT UNSIGNED | 是 | 点赞数 |
| sort_order | INT | 是 | 排序序号 |
| is_published | TINYINT(1) | 是 | 发布状态 |
| is_featured | TINYINT(1) | 是 | 是否精选 |
| seo_title | VARCHAR(200) | 否 | SEO标题 |
| seo_description | VARCHAR(500) | 否 | SEO描述 |

**索引说明**：

| 索引名 | 类型 | 字段 | 用途 |
|--------|------|------|------|
| PRIMARY | 主键 | id | 主键索引 |
| idx_date_sort | 普通 | date_sort | 按日期排序查询 |
| idx_year | 普通 | year | 按年份筛选 |
| idx_era_id | 普通 | era_id | 按年代筛选 |
| idx_sort_order | 普通 | sort_order | 按序号排序 |
| idx_is_published | 普通 | is_published | 筛选已发布 |
| idx_view_count | 普通 | view_count DESC | 浏览量排行 |
| ft_search | 全文 | title, description, historical_context | 全文搜索 |

### 4.3 photos（照片表）

**用途**：存储每个时间节点关联的历史照片。

```sql
CREATE TABLE `photos` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `node_id` BIGINT UNSIGNED NOT NULL COMMENT '关联节点ID',
  `url` VARCHAR(500) NOT NULL COMMENT '原图URL',
  `thumbnail_url` VARCHAR(500) NOT NULL COMMENT '缩略图URL',
  `medium_url` VARCHAR(500) COMMENT '中图URL',
  `large_url` VARCHAR(500) COMMENT '大图URL',
  `caption` VARCHAR(500) NOT NULL COMMENT '图片说明',
  `alt_text` VARCHAR(200) COMMENT '无障碍替代文本',
  `width` INT COMMENT '图片宽度(px)',
  `height` INT COMMENT '图片高度(px)',
  `file_size` INT COMMENT '文件大小(KB)',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序序号',
  `is_cover` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否封面图',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '软删除',
  PRIMARY KEY (`id`),
  INDEX `idx_node_id` (`node_id`),
  INDEX `idx_sort_order` (`sort_order`),
  INDEX `idx_is_cover` (`is_cover`),
```

### 4.6 messages（留言表）

```sql
CREATE TABLE `messages` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `nickname` VARCHAR(50) NOT NULL DEFAULT '匿名网友' COMMENT '昵称',
  `content` VARCHAR(500) NOT NULL COMMENT '留言内容',
  `content_filtered` VARCHAR(500) COMMENT '过滤后的内容',
  `ip_address` VARCHAR(45) NOT NULL COMMENT 'IP地址',
  `user_agent` VARCHAR(500) COMMENT 'User-Agent',
  `status` ENUM('pending', 'approved', 'rejected') NOT NULL DEFAULT 'pending' COMMENT '审核状态',
  `reject_reason` VARCHAR(200) COMMENT '拒绝原因',
  `reviewed_by` BIGINT UNSIGNED COMMENT '审核人ID',
  `reviewed_at` DATETIME COMMENT '审核时间',
  `like_count` INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '点赞数',
  `is_pinned` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否置顶',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '软删除',
  PRIMARY KEY (`id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_created_at` (`created_at` DESC),
  INDEX `idx_ip_address` (`ip_address`),
  INDEX `idx_reviewed_by` (`reviewed_by`),
  CONSTRAINT `fk_messages_reviewer` FOREIGN KEY (`reviewed_by`) REFERENCES `admins`(`id`) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='留言表';
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| nickname | VARCHAR(50) | 是 | 昵称，默认"匿名网友" |
| content | VARCHAR(500) | 是 | 原始留言内容 |
| content_filtered | VARCHAR(500) | 否 | 敏感词过滤后内容 |
| ip_address | VARCHAR(45) | 是 | IP地址（支持IPv6） |
| status | ENUM | 是 | pending待审核/approved通过/rejected拒绝 |
| reviewed_by | BIGINT UNSIGNED | 否 | 审核管理员ID |
| reviewed_at | DATETIME | 否 | 审核时间 |
| is_pinned | TINYINT(1) | 是 | 是否置顶展示 |

### 4.9 access_logs（访问日志表）

```sql
CREATE TABLE `access_logs` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `path` VARCHAR(500) NOT NULL COMMENT '访问路径',
  `method` VARCHAR(10) NOT NULL COMMENT 'HTTP方法',
  `status_code` INT NOT NULL COMMENT 'HTTP状态码',
  `ip_address` VARCHAR(45) NOT NULL COMMENT 'IP地址',
  `user_agent` VARCHAR(500) COMMENT 'User-Agent',
  `referer` VARCHAR(500) COMMENT '来源页面',
  `session_id` VARCHAR(100) COMMENT '会话ID',
  `node_id` BIGINT UNSIGNED COMMENT '访问的节点ID',
  `response_time` INT COMMENT '响应时间(ms)',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '访问时间',
  PRIMARY KEY (`id`),
  INDEX `idx_created_at` (`created_at`),
  INDEX `idx_path` (`path`(255)),
  INDEX `idx_ip_address` (`ip_address`),
```

## 5. 索引设计

### 5.1 索引设计原则

1. **高频查询字段建索引**：date_sort、era_id、status、created_at
2. **排序字段建索引**：sort_order、view_count
3. **搜索字段建全文索引**：title、description
4. **外键字段建索引**：node_id、era_id、tag_id
5. **避免过度索引**：单表索引不超过6个
