## 2. 用户浏览流程

### 2.1 首页访问流程

```
用户打开网站
    │
    ▼
浏览器请求 GET /
    │
    ▼
┌─────────────────────────────────────┐
│ Next.js SSG 返回预渲染首页 HTML       │
│ (首页为静态生成，直接返回)             │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 浏览器渲染首页                       │
│ - HeroSection 英雄区动画             │
│ - IntroSection 简介区               │
│ - EraNavigation 年代导航             │
│ - FeaturedTimeline 精选节点          │
│ - MessagePreview 留言预览            │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 客户端 SWR 请求精选数据              │
│ GET /api/v1/timeline?featured=true  │
│ GET /api/v1/messages?limit=6        │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ API 处理:                            │
│ 1. 检查 Redis 缓存                   │
│ 2. 缓存命中 → 返回缓存数据           │
│ 3. 缓存未命中 → 查询 MySQL           │
│ 4. 写入 Redis 缓存                   │
│ 5. 返回 JSON 数据                    │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 前端渲染数据                         │
│ - 展示精选时间节点                   │
│ - 展示最新留言                       │
└─────────────────────────────────────┘
```

## 3. 时间轴交互流程

### 3.1 时间轴加载流程

```
用户进入 /timeline 页面
    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 渲染页面骨架                          │
│    - TimelineHeader (标题+筛选器)        │
│    - TimelineContainer (空容器)          │
│    - TimelineSkeleton (骨架屏)           │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 2. 首次数据加载                          │
│    useInfiniteTimeline Hook 触发         │
│    GET /api/v1/timeline?page=1&pageSize=10│
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 3. API 处理                              │
│    a. 参数校验 (Zod)                     │
│    b. 检查 Redis 缓存 key:               │
│       timeline:page:{page}:size:{size}   │
│    c. 缓存命中 → 返回数据                │
│    d. 缓存未命中:                        │
│       - 查询 MySQL (Prisma)              │
│       - WHERE is_published=1             │
│       - ORDER BY date_sort ASC           │
│       - LIMIT pageSize OFFSET (page-1)*n │
│       - 关联查询 photos 表               │
│       - 写入 Redis (TTL=3600s)           │
│    e. 返回 JSON                          │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 4. 前端渲染                              │
│    a. 移除骨架屏                         │
│    b. 渲染 TimelineNode 组件             │
│    c. 触发 Framer Motion 入场动画        │
│    d. IntersectionObserver 监听节点可见性 │
│    e. 可见时播放淡入动画                  │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 5. 滚动加载更多                          │
│    a. ScrollTrigger 组件进入视口         │
│    b. 触发 loadMore()                    │
│    c. GET /api/v1/timeline?page=2...     │
│    d. 追加新节点到列表                   │
│    e. 重复直到 hasMore=false             │
└─────────────────────────────────────────┘
```

### 3.2 年代筛选流程

```
用户点击年代筛选标签 (如"延安岁月")
    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 更新 selectedEra 状态                 │
│ 2. 重置 useInfiniteTimeline              │
│    - 清空 nodes 列表                     │
│    - 重置 page=1                         │
│    - hasMore=true                        │
│ 3. 显示骨架屏                            │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 4. 请求筛选数据                          │
│    GET /api/v1/timeline?era=yanan        │
│    &page=1&pageSize=10                   │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 5. API 处理                              │
│    a. 验证 era 参数                      │
│    b. 查询 Redis:                        │
│       timeline:era:{era}:page:{page}     │
│    c. 查询 MySQL:                        │
│       WHERE era_id=? AND is_published=1  │
│       ORDER BY date_sort ASC             │
│    d. 返回数据                           │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 6. 渲染筛选结果                          │
│    - 滚动到顶部                          │
│    - 渲染新节点列表                      │
│    - 更新 URL: /timeline?era=yanan       │
└─────────────────────────────────────────┘
```

### 3.3 节点点击流程

```
用户点击时间轴上的节点卡片

## 4. 节点详情查看流程

### 4.1 详情弹窗交互流程

```
详情弹窗打开
    │
    ├─→ 用户点击大图
    │       │
    │       ▼
    │   打开 Lightbox 灯箱
    │       │
    │       ├─→ 滚轮缩放 (1x ~ 4x)
    │       ├─→ 拖动平移 (缩放>1时)
    │       ├─→ 左右键切换图片
    │       ├─→ ESC 关闭灯箱
    │       └─→ 点击外部关闭
    │
    ├─→ 用户点击"上一个"
    │       │
    │       ▼
    │   onNavigate('prev')
    │       │
    │       ▼
    │   加载上一个节点详情
    │       │
    │       ▼
    │   更新弹窗内容 (带过渡动画)
    │
    ├─→ 用户点击"下一个"
    │       │
    │       ▼
    │   onNavigate('next')
    │       │
    │       ▼
    │   加载下一个节点详情
    │       │
    │       ▼
    │   更新弹窗内容
    │
    ├─→ 用户按 ESC 键
    │       │
    │       ▼
    │   关闭弹窗
    │
    └─→ 用户点击关闭按钮/遮罩
            │
            ▼
        关闭弹窗，恢复背景滚动
```

### 4.2 键盘快捷键

| 按键 | 功能 |
|------|------|
| ESC | 关闭灯箱/弹窗 |
| ← | 上一个节点 |
| → | 下一个节点 |
| ↑ | 上一张图片 |
| ↓ | 下一张图片 |

## 5. 搜索流程

### 5.1 搜索建议流程

```
用户在搜索框输入文字
    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 输入防抖 (300ms)                      │
│ 2. 关键词长度 < 2 → 不请求               │
│ 3. 关键词长度 ≥ 2 → 请求建议             │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ GET /api/v1/search/suggest?q=延安        │
│                                          │
│ API 处理:                                │
│ a. 检查 Redis: search:suggest:{q}        │
│ b. 查询 MySQL:                           │
│    SELECT DISTINCT title FROM            │
│    timeline_nodes                        │
│    WHERE title LIKE '%延安%'             │
│    AND is_published=1                    │
│    LIMIT 10                              │
│ c. 写入 Redis (TTL=600s)                 │
│ d. 返回建议列表                          │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 展示搜索建议下拉框                       │
│ - 显示匹配的标题建议                     │
│ - 用户点击建议 → 直接搜索                │
└─────────────────────────────────────────┘
```

### 5.2 搜索执行流程

```
用户提交搜索 (回车/点击搜索按钮)
    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 跳转到 /search?q=延安                 │
│ 2. 记录搜索历史 (localStorage)           │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ GET /api/v1/search?q=延安&page=1         │
│                                          │
│ API 处理:                                │
│ a. 参数校验                              │
│ b. 检查 Redis: search:{q}:page:{page}    │
│ c. 查询 MySQL (全文搜索):                │
│    SELECT * FROM timeline_nodes          │
│    WHERE MATCH(title, description,       │
│      historical_context)                 │
│      AGAINST('延安' IN NATURAL MODE)     │
│    AND is_published=1                    │
│    ORDER BY relevance DESC               │
│    LIMIT 20                              │
│ d. 写入 Redis (TTL=600s)                 │
│ e. 返回搜索结果                          │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 渲染搜索结果                             │
│ - 显示结果数量                           │
│ - 渲染结果列表 (高亮关键词)              │
│ - 无结果时显示空状态                     │
└─────────────────────────────────────────┘
```

## 6. 留言提交与审核流程

### 6.1 留言提交流程

```
用户在留言页填写留言
    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 前端校验                              │
│    a. 内容不能为空                       │
│    b. 内容长度 ≤ 200字                   │
│    c. 昵称长度 ≤ 20字 (可选)             │
│    d. 频率限制 (前端):                   │
│       - 同一浏览器 5分钟内最多1条         │
└──────────────────┬──────────────────────┘
                   │ (校验通过)
                   ▼
┌─────────────────────────────────────────┐
│ 2. 提交留言                              │
│    POST /api/v1/messages                 │
│    Body: { nickname, content }           │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 3. API 处理                              │
│    a. 参数校验 (Zod)                     │
│    b. 获取客户端 IP 地址                  │
│    c. 频率限制检查 (Redis):              │
│       - key: msg_limit:{ip}              │
│       - 1小时内最多5条                   │
│       - 超限返回 429                     │
│    d. 敏感词检测:                        │
│       - level=2 (拒绝) → 返回错误        │
│       - level=1 (替换) → 替换为*         │
│    e. 写入 MySQL messages 表:            │
│       - status='pending'                 │
│       - content_filtered=过滤后内容      │
│    f. 返回成功消息                       │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 4. 前端展示结果                          │
│    - 显示"留言已提交，审核通过后展示"     │
│    - 清空表单                            │
└─────────────────────────────────────────┘
```

### 6.2 留言审核流程

```
管理员进入后台留言管理页
    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 加载待审核留言列表                    │
│    GET /api/v1/admin/messages            │
│    ?status=pending&page=1                │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 2. 管理员查看留言内容                    │
│    - 显示原始内容                        │
│    - 显示过滤后内容 (如有差异)           │
│    - 显示 IP、时间等信息                 │
└──────────────────┬──────────────────────┘
                   │
           ┌───────┴───────┐
           ▼               ▼
    ┌──────────┐    ┌──────────┐
    │ 通过审核  │    │ 拒绝审核  │
    └─────┬────┘    └─────┬────┘
          │               │
          ▼               ▼
┌─────────────────┐ ┌─────────────────┐
│ PATCH /api/v1/  │ │ PATCH /api/v1/  │
│ admin/messages/ │ │ admin/messages/ │
│ {id}/approve    │ │ {id}/reject     │
│                 │ │ Body: {reason } │
└────────┬────────┘ └────────┬────────┘
         │                   │
         ▼                   ▼
┌─────────────────┐ ┌─────────────────┐
│ API 处理:        │ │ API 处理:        │
│ - 更新 status   │ │ - 更新 status   │
│   ='approved'   │ │   ='rejected'   │
│ - 记录 reviewed │ │ - 记录 reason   │
│   _by/_at       │ │ - 记录 reviewed │
│ - 清除留言缓存  │ │   _by/_at       │
└────────┬────────┘ └────────┬────────┘
         │                   │
         ▼                   ▼
┌─────────────────┐ ┌─────────────────┐
│ 留言在前台展示  │ │ 留言不展示      │
└─────────────────┘ └─────────────────┘
```

## 7. 管理员登录流程

### 7.1 登录流程

```
管理员访问 /admin/login
    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 输入用户名和密码                      │
│ 2. 前端校验非空                          │
│ 3. 提交表单                              │
│    POST /api/v1/auth/login               │
│    Body: { username, password }          │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 2. API 处理                              │
│    a. 参数校验                           │
│    b. 获取客户端 IP                      │
│    c. 登录限流检查:                      │
│       - key: login_limit:{ip}            │
│       - 30分钟内最多5次失败              │
│       - 超限返回 429                     │
│    d. 查询管理员:                        │
│       SELECT * FROM admins               │
│       WHERE username=? AND status='active'│
│    e. 验证密码 (bcrypt.compare)          │
│    f. 密码错误:                          │
│       - INCR login_limit:{ip}            │
│       - INCR failed_login_count          │
│       - 失败5次锁定账户                  │
│       - 返回 401                         │
│    g. 密码正确:                          │
│       - 生成 JWT Token (24h过期)         │
│       - 更新 last_login_at/ip            │
│       - 重置 failed_login_count          │
│       - 清除 login_limit:{ip}            │
│       - 设置 HttpOnly Cookie             │
│       - 返回用户信息                     │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│ 3. 登录成功                              │
│    - 跳转到 /admin/dashboard             │
│    - 存储 Token 到 Cookie                │
└─────────────────────────────────────────┘
```

### 7.2 登录失败处理

```
登录失败
    │
    ├─→ 用户名不存在
    │       │
    │       ▼
    │   返回 "用户名或密码错误"
    │   (不提示具体是哪个错误，防止枚举)
    │
    ├─→ 密码错误
    │       │
    │       ▼
    │   INCR login_limit:{ip} (TTL=1800s)
    │   INCR admin.failed_login_count
    │       │
    │       ├─→ failed_login_count < 5
    │       │       │
    │       │       ▼
    │       │   返回 "用户名或密码错误"
    │       │       "剩余尝试次数: {5-count}"
    │       │
    │       └─→ failed_login_count >= 5
    │               │
    │               ▼
    │           锁定账户 (locked_until=30min后)
    │           更新 status='locked'
    │           返回 "账户已锁定，请30分钟后重试"
    │
    ├─→ 账户已锁定
    │       │
    │       ▼
    │   返回 "账户已锁定，请{剩余时间}后重试"
    │
    ├─→ 账户已禁用
    │       │
    │       ▼
    │   返回 "账户已禁用，请联系管理员"
    │
    └─→ IP 频率超限
            │
            ▼
        返回 429 "尝试次数过多，请稍后再试"
```

    │
    ├─→ 连接失败
    │       │
    │       ▼
    │   重试3次 (指数退避)
    │   仍失败 → 返回 503
    │
    ├─→ 死锁
    │       │
    │       ▼
    │   自动重试
    │   记录日志
    │
    ├─→ 唯一键冲突
    │       │
    │       ▼
    │   返回 409 "数据已存在"
    │
    ├─→ 外键约束冲突
    │       │
    │       ▼
    │   返回 400 "关联数据不存在"
    │
    └─→ SQL 语法错误
            │
            ▼
        记录错误日志
        返回 500
        告警通知
```

## 附录：状态码定义

| 状态码 | 含义 | 使用场景 |
|--------|------|---------|
| 200 | 成功 | 请求成功 |
| 201 | 创建成功 | POST 创建资源 |
| 204 | 无内容 | DELETE 成功 |
| 400 | 参数错误 | 请求参数校验失败 |
| 401 | 未认证 | 未登录或Token失效 |
| 403 | 无权限 | 权限不足 |
| 404 | 不存在 | 资源未找到 |
| 409 | 冲突 | 唯一键冲突 |
| 422 | 不可处理 | 业务逻辑校验失败 |
| 429 | 限流 | 请求过于频繁 |
| 500 | 服务器错误 | 内部异常 |
| 503 | 不可用 | 服务维护中 |
