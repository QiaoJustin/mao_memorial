4. 生成完整的数据库插入语句，确保数据一致性

## 三、图片命名规则

### 3.1 命名格式

```
{年份}-{月份}-{地点拼音}-{序号}.jpg
```

**示例**：
- `1913-00-长沙-01.jpg`（1913年，无具体月份，长沙，第1张）
- `1936-00-保安-01.jpg`（1936年，无具体月份，保安，第1张）
- `1945-08-重庆-01.jpg`（1945年8月，重庆，第1张）
- `1949-10-北京-01.jpg`（1949年10月，北京，第1张）

### 3.2 规则说明

| 组成部分 | 格式 | 说明 |
|----------|------|------|
| 年份 | 4位数字 | 如 1913、1949 |
| 月份 | 2位数字 | 具体月份用实际值，无月份用 `00`，如 `03`、`08`、`00` |
| 地点拼音 | 小写字母 | 中文地点转拼音，如 `changsha`、`beijing`、`yanan` |
| 序号 | 2位数字 | 同时间同地点多张图片时递增，如 `01`、`02` |

## 六、风险与注意事项

### 6.1 风险

1. **地点识别错误**：部分描述中地点不明确，可能导致命名不准确
2. **文件名冲突**：同时间同地点多张图片需确保序号正确
3. **中文编码问题**：文件名中不使用中文，全部使用拼音

### 6.2 注意事项

1. 保留原始图片文件不变，只进行复制操作
2. 确保 Next.js `public` 目录下的文件可被静态资源路由访问
3. 导入前先清空数据库中已有的测试数据
4. 图片URL格式：`/timeline/{文件名}.jpg`
